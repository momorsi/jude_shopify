Which web stack for the admin UI?

This runs on the Windows sync server; the business team accesses it from the office network

A

FastAPI + HTMX & Bootstrap

Server-rendered HTML with lightweight JS sprinkles. No build step, no Node.js. Pages feel fast and interactive without a full SPA. Fits cleanly inside the existing Python repo.

Pros

  • Pure Python, zero JS toolchain
  • Fastest to build
  • Easy to maintain
  • Auto-refreshing tables & logs via HTMX polling

Cons

  • Less "app-like" feel
  • Complex interactions harder to implement
B

FastAPI + React (Vite)

Full single-page app frontend served from the same Python backend. Rich interactive UI — sortable tables, live charts, modal dialogs, real-time log streaming.

Pros

  • Most polished UX
  • Easy live updates (WebSocket/SSE)
  • Charts & data tables built-in
  • Separate frontend codebase scales well

Cons

  • Requires Node.js + build step
  • More setup complexity
  • Bigger scope to maintain
C

Streamlit

Python-only dashboard framework. Spin up a data app in hours — charts, tables, forms, filters — all in pure Python. Best for internal ops tools and analytics.

Pros

  • Fastest possible build time
  • Native charts & dataframes
  • Zero HTML/CSS needed
  • Great for ODBC query results

Cons

  • Limited custom styling
  • Not ideal for forms/config editing
  • Heavier runtime dependency
